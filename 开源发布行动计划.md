## 你需要手动完成的工作

### 🚨 优先级最高：推送代码到 GitHub

推送被阻止了，有两个问题需要解决：

**问题 1：分支保护规则阻止了 force-push**

1. 打开 https://github.com/AetherCore-Dev/ag402/settings/rules
2. **临时禁用** main 分支的保护规则（或删除当前的 ruleset）
3. 如果仓库之前有初始化内容（比如 GitHub 自动生成的 README），**建议直接删除仓库并重建**：
   - Settings → General → 最底部 Danger Zone → Delete this repository
   - 重新创建：New repository → 名字 `ag402` → Public → **不勾选任何初始化选项**
   - 这样最干净

**问题 2：OAuth App 缺少 `workflow` scope**

当前的 git credential（OAuth token）没有修改 GitHub Actions workflow 文件的权限。解决方法：

**方法 A（推荐）：使用 SSH Key**
```bash
# 1. 生成 SSH Key（如果还没有的话）
ssh-keygen -t ed25519 -C "你的邮箱"

# 2. 复制公钥
cat ~/.ssh/id_ed25519.pub | pbcopy

# 3. 添加到 GitHub: https://github.com/settings/keys → New SSH key → 粘贴

# 4. 更换远程仓库地址为 SSH
cd /Users/allenenli/Documents/Agent_Payment_Protocol
git remote set-url origin git@github.com:AetherCore-Dev/ag402.git

# 5. 推送
git push -u origin main
```

**方法 B：重新授权 HTTPS token**
```bash
# 1. 删除旧的 credential
git credential-osxkeychain erase
# 输入：host=github.com  protocol=https  然后按两次回车

# 2. 创建 Personal Access Token (PAT)
#    打开 https://github.com/settings/tokens?type=beta
#    → Generate new token → 勾选 repo + workflow scope
#    → 复制 token

# 3. 推送时粘贴 token 作为密码
git push -u origin main
# Username: 你的 GitHub 用户名
# Password: 粘贴 PAT token
```

**注意：** 如果你重建了空仓库并解决了 workflow 权限，执行：
```bash
cd /Users/allenenli/Documents/Agent_Payment_Protocol
git push -u origin main
```

如果你没有重建仓库（仍然有旧内容），执行：
```bash
cd /Users/allenenli/Documents/Agent_Payment_Protocol
git push --force -u origin main
```
